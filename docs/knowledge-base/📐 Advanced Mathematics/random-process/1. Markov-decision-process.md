## 1. Preliminaries 
### (1) Counting Measure 
We define the Counting Measure[^8] in mathematics, the counting measure can be defined on any measurable space (for example, the set $X$ on any Sigma-algebra, see <span class="equation-citator-citation equation-citator-cite-callout" data-ec-kind="def" data-ec-refs='[{&quot;file&quot;:&quot;📐 Advanced Mathematics/combinatorics/1. Sigma-algebra and Lebesgue measure.md&quot;,&quot;crossFile&quot;:&quot;1&quot;,&quot;tag&quot;:&quot;sigma-algebra&quot;}]' style="cursor: default; color: #4199df;">Definition. <span style="text-decoration: none; cursor: pointer; color: #4199df;">sigma-algebra</span></span>[^1]), but it's **often in a countable sets**. 

In combinatorics theory, we denote the **ground set** as $X$, then the ==**sigma-algebra**== $\Sigma$ defined by such ground set $X$ is given by : 
$$
\Sigma  \subset \mathbb{P}(X) \tag{1.1.1}
$$
Where $\mathbb{P}(X)$, or $2^{X}$ means the power set of $X$[^2] (**to be more clean, all of the  subsets**). 

After that, the $(X, \Sigma)$ forms a positive measure space $\Sigma \rightarrow  [0, +\infty]$. Then the counting measure is only $\sigma$-finite <b><mark style='background: transparent; color: orange'>if and only if the space</mark></b> $X$ is countable. 

Then, the **counting measure is the positive measure $\Sigma \to [0, +\infty]$, defined by** :  
$$
\mu (A) = \begin{cases}
|A|  \qquad \text{if} \quad  A  \text{ is a finite set}  \\
+\infty\qquad \text{if} \quad A \text{ is an infinite set}
\end{cases} \tag{1.1.2}
$$
for all $A \in \Sigma$, where $|A|$ is the **cardinality**(number of elements) of set $A$. 

> [!NOTE] Notations 
> We often use $(X, \Sigma, \mu)$  to denote a space. For this part, the first parameter is **ground set**, second is its subsets. Third (optional) is the measure.
> 

### (2) Integration on the counting measure  
#### 1. Discrete representation 
We take the measure space $(\mathbb{N}, 2^{\mathbb{N}}, \mu)$, Take **any measurable function** $f : \mathbb{N} \to [0, + \infty]$. it can be ==**represented pointwise**== as follows : 
$$
f(x )  = f(1) + f(2) + \dots   f (n) + \dots +  f(\infty) \tag{1.2.1}
$$
then we got :  
$$
f(x )  = \sum_{n= 1}^{\infty}  f(n ) 1_{\{  n\}} (x) = \lim_{M \rightarrow  \infty } \underbrace{ \sum_{n = 1}^{M}f(n ) 1_{\{  n\}} (x)  }_{ \phi_{M} (x) } \tag{1.2.2}
$$
where we represent the $1_{\{ n\}}$ as : 
$$
\boldsymbol{1}_{\{n\}} (x) = \begin{cases}
1  &  \text{x= n}  \\
0  &  \text{otherwise}
\end{cases} \tag{1.2.3}
$$
Here, $\{n\}$ **means a singleton with element $n$**, and each $\phi_{M}$ is measurable. apparently, $\phi_{M+1}(x) = \phi_{M} + f(M +1) \geq \phi_{M} (x)$  

#### 2. Integration of function over countable measure 
From <span class="equation-citator-citation equation-citator-cite-equation" data-ec-kind="eq" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;1.2.2&quot;}]' style="cursor: default; color: #4199df;">(<span style="text-decoration: none; cursor: pointer; color: #4199df;">1.2.2</span>)</span>, we know a function $f(x)$ can be represented with $\phi_{M} (x)$, i.e., : 
$$
f (x) = \lim_{M \rightarrow \infty} \phi_{M} (x) \qquad  \phi_{M} (x) = \sum_{n =1}^{M} f(n) \boldsymbol{1}_{\{ n\}} (x) \tag{1.2.4}
$$
here, since $\mu$ is <b><mark style='background: transparent; color: orange'>countable measure</mark></b>, **we have** $\mu(\{n\}) = 1$, so, for any measure $\mu$ on the space $(X, \Sigma , \mu)$ :  
$$
\int_{\mathbb{N}} \phi_{M} d \mu  =  \sum_{n = 1}^{M} f(n) \int_{N} \boldsymbol{1}_{\{ n\}} (x) d\mu = \sum _{n = 1}^{M} f(n) \mu (\{ n\}) \tag{1.2.5}
$$
So we have <b><mark style='background: transparent; color: red'>the integral over the countable measure</mark></b> is :  
$$
\boxed{\int _{\mathbb{N}} f(x) d \mu = \sum_{n = 1}^{\infty} f(n)} \tag{1.2.6}
$$
We note that, <b><mark style='background: transparent; color: orange'>this integration also</mark></b> defines a measure $\tilde{\mu}$ on $(X, \Sigma)$ via :
$$
\tilde{\mu} (A) := \sum_{a \in  A} f(a) \qquad  \forall  A \in  X \tag{1.2.7}
$$
the $\tilde{\mu}$ is a new  measure on $A$, with taking $f(x) = 1$ gives counting measure.

## 2. Markov-decision-process (MDP) 
### (1) Definition of MDP 
The reinforced learning is  a model for a system evolves  as different actions applied to the system.
For a <b><mark style='background: transparent; color: red'>Markov decision process (MDP)</mark></b>[^3][^4], that defined by the following 4-element tuple : 
$$
\Large \boxed{\text{MDP: } \left( \mathcal{S}, \mathcal{A}, P_{a}, R_{a} \right) } \tag{2.1.1}
$$
This means : <b><mark style='background: transparent; color: red'>available state, available action, model decision and reward</mark></b> 

We describe the **Markov decision process** as a 4-tuple process where : 
1. $\mathcal{S}$ is a set of states called <b><mark style='background: transparent; color: red'>state space</mark></b> 
2. $\mathcal{A}$ is a set of actions called **action space**, and $A_{s}$ often refers to the <b><mark style='background: transparent; color: red'>set of actions available</mark></b> from state $\mathcal{S}$. 
3. $P_{a}$ is a **intuitive level**, or called $P_{a} (s, s')$. For the action $a$ in state $s$ will lead to $s'$ at the time $t + 1$.  <b><mark style='background: transparent; color: orange'>This is defined to satisfy following relation</mark></b> : 
$$
\text{Pr}( s_{t+1} \in S' | s_{t} = s, a_{t} = a) \overset{\text{action determined}}{\longrightarrow} \int_{S'} P_{a} (s, s') ds' \tag{2.1.2}
$$
$\text{Pr}$ describes the ==probability that what probability the next state to reach, under certain state and actions==. This is integral for every $S' \subset S$ measurable. 
4.  $R_{a}$ is <b><mark style='background: transparent; color: red'>immediate reward</mark></b>, or **expected immediate reward** received after action $a$ is taken. and such reward is a general random variable. 

> [!NOTE] 
> $P_{a}(s, s')$ is the transition of the system 
> $R_{a} (s, s')$ often is $1$ when the state after transition is correct. 

### (2) Simple example 
 <b><mark style='background: transparent; color: red'>A simple example is the robot navigation system</mark></b> (source: [^9]): 
 
![[📐 Advanced Mathematics/random-process/assets/Pasted image 20260701144758.png|title:robotics navigation system|fig:2.1|254]]

For the **above robotics navigation task** shown in <span class="equation-citator-citation equation-citator-cite-figure" data-ec-kind="fig" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;2.1&quot;}]' style="cursor: default; color: #4199df;">Fig. <span style="text-decoration: none; cursor: pointer; color: #4199df;">2.1</span></span>, we have :   
1. $\mathcal{S}$ is the set of states in the MDP, as the navigation gridworld shows. 
2. $\mathcal{A}$ is the actions that robot can take at each state, e.g., “go forward”, “turn right”, “turn left”, “stay at the same location”. 
3. If we don't know that the robot moves exactly but only up to some approximation, this amounts to defining a **transition function** that from current state $s$ to a new state $s'$. 
$$T : \mathcal{S} \times  \mathcal{A} \times \mathcal{S} \to [0,1] \tag{2.2.1} $$
 such that it's condition probability (here for example $\text{forward} \to T = 0.8$, $\text{turn right} \to T =0.1$, so we said $\to [0,1]$)
$$T(s, a, s') = P(s' | s, a) \tag{2.2.2} $$
And the **transition function is a probability distribution** that $\sum_{s' \in  \mathcal{S}} T(s,a, s') = 1$  for all $s \in  \mathcal{S}$ and $a \in \mathcal{A}$. So in many cases, like <span class="equation-citator-citation equation-citator-cite-equation" data-ec-kind="eq" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;2.1.1&quot;}]' style="cursor: default; color: #4199df;">(<span style="text-decoration: none; cursor: pointer; color: #4199df;">2.1.1</span>)</span>, we use $P$ instead of $T$ here. 

We can construct a notion using the concept of <b><mark style='background: transparent; color: orange'>reward</mark></b> $r  : \mathcal{S} \times \mathcal{A} \rightarrow \mathbb{R}$,  we can say that **the robot gets  a reward** $r (s, a)$ if robot takes an action $a$ at state $s$. Such reward is designed by the user.  

### (3) Return Factor and Discount Factor  
For the MDP in <span class="equation-citator-citation equation-citator-cite-equation" data-ec-kind="eq" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;2.1.1&quot;}]' style="cursor: default; color: #4199df;">(<span style="text-decoration: none; cursor: pointer; color: #4199df;">2.1.1</span>)</span>, the continues taking actions to result in a <b><mark style='background: transparent; color: red'>trajectory</mark></b> : 
$$
\tau = (s_{0}, a_{0}, r_{0}, s_{1}, a_{1}, r_{1}, \dots ) \tag{2.3.1}
$$
For each time step $t$, the robot is at state $s_{t}$, then we define the <b><mark style='background: transparent; color: blue'>return of the trajectory as</mark></b> : 
$$
R(\tau) = r_{0} + r_{1} + r_{2} + \dots \tag{2.3.2}
$$
we also introduce the <b><mark style='background: transparent; color: red'>discount factor</mark></b> $\gamma$, which is often defined by the **discount rate** $r$ :  
$$
\gamma = \frac{1}{1 + r} \tag{2.3.3}
$$
A **lower discount factor makes the decision maker more short-sighted** (greedy for recent best solutions). Then the discounted return, or <b><mark style='background: transparent; color: red'>"reward"</mark></b> here, such **reward is defined as** :
$$
\boxed{R(\tau) = \sum_{t =0}^{\infty} \gamma^{t}  R_{a_{t}} } \tag{2.3.4}
$$
And for a large value of the discount factor, e.g. $\gamma = 0.99$, then robot is **encouraged to explore more and find the best trajectory**.  A successful value often between $\gamma \in  [0.9, 0.99]$ 

### (4) Optimization Policy & Objective definition
The <b><mark style='background: transparent; color: orange'>reinforcement learning problem</mark></b> is typically modeled using **Markov Decision Processes**. We consider that we want the reach a specific final goal. So <b><mark style='background: transparent; color: red'>we determine state of the result</mark></b> as $\pi (x)$ by a <b><mark style='background: transparent; color: red'>given policy</mark></b>. The objective as to <b><mark style='background: transparent; color: red'>maximize the cumulative function of the random rewards</mark></b> :
$$
\eta (\pi) = \mathbb{E}\left[ \sum_{t = 0}^{\infty} \gamma^{t} R_{a_{t}} (s_{t}, s_{t+ 1 })\right] \tag{2.4.1}
$$
where $a_{t} = \pi(s_{t})$, or <b><mark style='background: transparent; color: red'>Actions at each time step is defined given by the policy</mark></b>. 

We note another objective function to use is the $H$, or the <b><mark style='background: transparent; color: orange'>step return</mark></b>; 
$$
\mathbb{E}\left[ \sum_{t= 0}^{H-1} R_{a_{t}} (s_{t}, s_{t + 1})\right] \tag{2.4.2}
$$

## 3.  Bellman Optimality Equation
### (1) For a determined-state problem 
Firstly, **we represent** : 
1. policy value function as $V_{\pi} (s)$
2. Optimal value function as $V^{*} (s)$   (We want to make policy function to approach this)

Bellman Equation[^6] helps <b><mark style='background: transparent; color: red'>evaluate the expected reward relative to the advantage or disadvantage of each state</mark></b>. the **Value function is defined as** : 
$$
\boxed{\Large  V^{*}(s ) =  \max_{a} \left[ R (s) + \gamma  V^{*} (s')  \right] } \tag{3.1.1}
$$
here, $R$ is period-specific objective function, $V (s')$ is the future objective function. 

> [!hint] 
> Sometimes, if $R$ is related with $a$ we use : 
> $$ V^{*}(s) =  \max_{a} \left[ R (s, a) + \gamma  V^{*} (s') \right]$$

> Bellman showed that a dynamic [optimization](https://en.wikipedia.org/wiki/Optimization_\(mathematics\) "Optimization (mathematics)") problem in [discrete time](https://en.wikipedia.org/wiki/Discrete_time "Discrete time") can be stated in a [recursive](https://en.wikipedia.org/wiki/Recursion "Recursion"), step-by-step form known as [backward induction](https://en.wikipedia.org/wiki/Backward_induction "Backward induction") by writing down the relationship between the value function in one period and the value function in the next period.  

Here, $\max_{a}$ means the choice of the best possible action. This means, <b><mark style='background: transparent; color: orange'>to choose the best possible action, the value is the current response + weighted value of future possible states</mark></b>. <b><mark style='background: transparent; color: red'>For a more general equation</mark></b>, we still need to consider the probability of next option. i.e. adding $P(s'|s)$  into <span class="equation-citator-citation equation-citator-cite-equation" data-ec-kind="eq" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;3.1.1&quot;}]' style="cursor: default; color: #4199df;">(<span style="text-decoration: none; cursor: pointer; color: #4199df;">3.1.1</span>)</span> : 
$$
\Large \boxed{V (s) = \max_{a} \left| R (s) + \gamma \sum_{s} P (s'|s, a) V(s') \right|} \tag{3.1.2}
$$
We note in application case, we use <span class="equation-citator-citation equation-citator-cite-equation" data-ec-kind="eq" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;3.1.1&quot;}]' style="cursor: default; color: #4199df;">(<span style="text-decoration: none; cursor: pointer; color: #4199df;">3.1.1</span>)</span> more than <span class="equation-citator-citation equation-citator-cite-equation" data-ec-kind="eq" data-ec-refs='[{&quot;file&quot;:null,&quot;crossFile&quot;:null,&quot;tag&quot;:&quot;3.1.2&quot;}]' style="cursor: default; color: #4199df;">(<span style="text-decoration: none; cursor: pointer; color: #4199df;">3.1.2</span>)</span> 

The often the ideal action is to <b><mark style='background: transparent; color: red'>take the reward from the end state back to the start</mark></b>, which means, the end state has been defined (as $\pi$ later). A **comprehensive picture is drawn as follows** (the $V$ is calculated by ):

The action is to bring us to the highest value function. So it's very easy to find the clean path. 

### (2) For A stochastic system 
For example, a consumer with initial wealth endowment $s_{0}$, he  also have instantaneous utility function $u(c)$ (this will **determine the objective depends by the consumption**, so this is deemed as "reward"), discounts the next period utility at a rate of $\beta$, and $c$ here denotes the consumption. We should choose a consumption plan, then the <b><mark style='background: transparent; color: red'>current interest rate on the left money in the bank</mark></b> is $r$. 

So the consumer should choose a sequence $\{ c_{t} \}$  <b><mark style='background: transparent; color: blue'>such a way that their lifetime expected utility is maximized</mark></b>, so we often use following function : 
$$
\max_{\{c_{t} \}_{t = 0}^{\infty}} \sum_{t = 0}^{\infty} \beta^{t} u(c_{t}) \tag{3.2.1}
$$
Then the constraint is  :  
$$
s_{t +1} = (1 + r)(s_{t} - c_{t}) \qquad  c_{t} > 0 \qquad  \lim_{ t \rightarrow  \infty} a_{t} \geq 0 \tag{3.2.2}
$$
the value function should be :   
$$
V(s) = \max_{0 \leq c \leq  a} \left\{ u(c) + \beta V((1 + r) (s_{t} - c )) \right\} \tag{3.2.3}
$$
