    更新日志 1.2.X

    版本 1.2.0

    版本 1.2.0 - 功能改进。

    🐛 已修复的 Bug：

    • 如果 enableInSourceMode 设置被禁用，自动补全现在不会在源码模式下显示。（在 1.1.0 的发布修订中修复）
    • 公式自动补全现在会在单行公式块的分隔符后添加空格。
    • 如果脚注没有提供别名（标签），现在将使用文件名作为别名。
    • 修复教程中的一些空行问题，并为新功能添加教程。
    • 重构公式工具函数，将多个函数合并为一个。

    发布补丁：提供一个选项来启用/禁用在自动编号公式时更新引用。

    ✨ 新功能：

    1. 链接跳转：双击跳转到公式。Ctrl + 双击创建相邻窗口并跳转到公式。（仅支持实时预览模式）

    2. 重命名标签：现在可以选择标签并右键重命名。引用将自动更新。

    3. 公式自动编号：公式自动编号现在会自动更新相关引用以确保编号一致。

    版本 1.2.1

    版本 1.2.1 - Bug 修复和小改进。

    🐛 已修复的 Bug：

    • 修复代码块前有空格时自动编号可能无法正常工作的问题
    • 修复公式跳转问题 BUG #6，现在 Ctrl + 双击可以正确跳转。
    • 当存在多个视图且悬停在非活动视图上时，双击现在也能正确跳转（同样是 BUG #6）。
    • 修复在开启「删除未使用标签」时重命名标签可能出现的删除 Bug（同时提高了重命名标签的效率）
    • 修复自动编号公式时原文件中引用未正确更新的 Bug。
    • 修复教程中的一些公式引用问题。

    发布补丁：修复 Bug——如果显示重复提示，重命名标签会失败

    🚀 优化：

    • 自动编号和标签重命名现在显示已重命名引用的信息。

    ✨ 新功能：

    • （实验性功能）在标注中使用内联代码块引用：添加此功能以在标注块中渲染引用。（针对 Bug #4）。

    版本 1.2.2

    版本 1.2.2 - 多项关键 Bug 修复和用户体验的重大改进。

    🐛 已修复的 Bug 和优化：

    • （关键 Bug 修复） 修复 Bug #14，现在即使使用单行标签，当前文件也能正确自动编号。（笔记在自动编号后不会再乱成一团😅）
    • 修复 Bug #15，引用现在会在实时编辑器预览组件中渲染。
    • 修复 Bug #16，自动补全现在能正常工作，不会创建多余的空格
    • 修复 Bug #17，在中间位置补全时引用建议将正常工作。
    • 修复多个相同公式的渲染处理，现在会自动合并，$\ref{eq:1.1.1, 1.1.1, 1.1.2, 1.1.2, 1.1.3, 1.1.4, 1.1.6, }$ 现在应该渲染为 (1.1.1~4), (1.1.6)，而在之前的版本中渲染为 (1.1.1~2), (1.1.2~4), (1.1.6)。
    • 修复当 ref 括号和引用前缀之间有空格时引用无法正确渲染的问题（例如 ref{ eq:1.1} 无法正确渲染）
    • 修复 PDF 导出问题：Markdown 会添加路径前缀而不是在相同文件夹下创建。
    • enableAutoNumberGlobalPrefix 设置未被使用，切换它没有效果。现在此设置已移除。

    ✨ 新功能：

    • （重大改进） 跨文件引用现在支持在引用中使用内部括号（为此进行了大规模重构，PDF 支持和自动补全也已更新以支持此功能）。
    • 优化教程和 Readme 文件，添加新功能，移除不必要的部分和重复内容。将规则部分从 Readme 移至教程。
    • （小幅改进） 优化用户体验：如果在引用后输入空格，插件将停止建议。
    • （用户体验改进） 添加插入标签并自动编号的新命令。（允许更轻松地快速插入标签）
    $\ref{eq:2^{1.1}}$    // 引用中使用内部括号

    版本 1.2.3

    版本 1.2.3 - 改进和一些小 Bug 修复。

    🐛 已修复的 Bug：

    • 修复 Bug #24，现在插入命令通过插入内容来替换选区。

    • 现在支持更好的脚注识别（以纯脚注格式书写 # 和 ^）。

    • 修复样式问题 glitch #23

    🚀 改进：

    • 当标签重复时，自动编号仅更新第一个标签出现的引用，这使得您可以将包含重复标签的部分追加到笔记中，然后正常使用自动编号。

    • 在公式中插入标签时更精确的标签定位。

    • 在 Readme 中添加如何安装此插件的教程。

    • （主要改进） 在此版本中使用生产模式构建插件（使用 npm run build），减小插件体积并提高性能，插件大小从 593kb 减少到 73kb）

    ✨ 新功能：

    • 引用文件上标现在在悬停时会在弹出窗口中渲染脚注。

    • 在上标弹出窗口渲染中添加对纯文本和网页链接格式脚注的支持

    • 在设置标签页中自定义公式弹出窗口的宽度

    🛠️ 发布补丁：

    • 现在禁止在设置中使用大括号 {} 作为分隔符。

    🐛 已修复的 Bug：

    • （关键） 修复 Critical #35，当没有标签可重命名且启用了删除未使用标签时，插件将正常删除未使用的标签😅。

    版本 1.2.5

    版本 1.2.5 - Bug 修复和一些改进。

    🐛 已修复的 Bug：

    • 修复自动编号功能：在 1.2.4 中，自动编号仍可能错误地推送第二和第三个标签重命名的情况，现在它将始终正确更新仅第一个标签出现。
    • （关键） 修复引用更新问题 Bug #53，自动编号后当前文件中的引用现在将正确更新。
    • （关键） 修复引用更新问题 Bug #54，删除未使用的引用现在仅影响同一文件的引用。
    • 在重命名标签组件中按 Enter 键现在将确认重命名标签。
    • 更好的前缀设置检查（不允许大括号和美元符号或留空），同时美元符号现在在分隔符中被禁止，因为它不安全。
    • 修复样式问题 #51，添加边距，现在将别名最大渲染宽度限制为 380px。
    • 修复 PDF 引用颜色的描述 #49
    • 修复转义美元符号问题 #40

    🚀 增强：

    • 添加增强 #52，现在默认使用「多引用分隔符 + 空格」（在设置标签页中设置）以获得更好的渲染效果。
